Dívida de Design/UX

Categoria auditada: Dívida de Design/UX

Repositório: Birth Hub Innovation 360

Escopo auditado: Componentes de agentes no front-end canônico, tokens visuais globais e guardrails de consistência de estilo já adotados pelo projeto.

Quantidade de itens comprovados: 1

Status final da análise: Achados comprovados

Data/hora da análise: 2026-04-14T16:11:14.638Z

Método

A auditoria comparou o design system efetivo do app (`globals.css`) com componentes de agentes que materializam apresentação inline e duplicação local. Também foram considerados os guardrails já codificados para evitar novas ocorrências.

Resultado

Superfícies de agentes contornam o design system com estilo inline

Diagnóstico: O app já possui tokens globais de cor, tipografia e superfície. Mesmo assim, as telas de agentes concentram dezenas de estilos inline e blocos visuais duplicados. A consistência visual fica dependente de edição manual por componente.

Evidências:
  • apps/web/app/globals.css:1-12 define tokens globais como `--font-sans`, `--surface`, `--text`, `--accent`, `--card` e `--border`.
  • scripts/ci/check-web-inline-style-freeze.mjs:5-10 protege apenas quatro arquivos específicos contra novos `style={` e orienta mover apresentação para CSS.
  • scripts/ci/check-web-inline-style-freeze.mjs:61-69 declara explicitamente que novos inline styles devem ir para CSS compartilhado.
  • apps/web/components/agents/PolicyManager.tsx` e `apps/web/components/agents/agent-detail-tabs.tsx` somam 71 ocorrências de `style={{` na inspeção textual.
  • artifacts/quality/jscpd/jscpd-report.json registra `PolicyManager.tsx` com 87.5% de linhas duplicadas e `agent-detail-tabs.tsx` com 54.84%.

O Juro (Impacto): Pequenos ajustes de UX, tema e consistência visual custam mais do que deveriam. O front-end passa a ter ilhas visuais que não evoluem no mesmo ritmo do design system central.

Plano de Quitação:
  1. Levar os padrões repetidos de card, seção, tabela e detalhe para classes ou primitivas compartilhadas.
  2. Ampliar o guardrail de inline style para incluir as superfícies de agentes que já concentram dívida.
  3. Revalidar as telas de agentes em desktop e mobile após a extração.
Prompt de Execução/Correção 
Papel
Você é um engenheiro de front-end responsável por alinhar superfícies críticas ao design system existente.

Missão
Reduzir estilos inline e duplicação visual nas superfícies de agentes.

Escopo
Atue somente nos componentes de agentes citados e nos estilos compartilhados estritamente necessários.

Arquivos-alvo
- apps/web/components/agents/PolicyManager.tsx
- apps/web/components/agents/agent-detail-tabs.tsx
- apps/web/app/globals.css
- scripts/ci/check-web-inline-style-freeze.mjs

Correções obrigatórias
- Mapear os padrões visuais repetidos antes de editar.
- Mover estilos repetidos para classes ou primitivas alinhadas aos tokens globais.
- Atualizar o guardrail de inline style para cobrir as superfícies de agentes corrigidas.

Restrições
- Não criar um novo design system paralelo.
- Não alterar o comportamento funcional das telas de agentes.
- Não introduzir inline styles novos durante a correção.

Critérios de aceite
- Os componentes de agentes passam a reutilizar tokens e estilos compartilhados.
- A quantidade de apresentação duplicada cai de forma verificável.
- O guardrail impede regressão do mesmo problema.
