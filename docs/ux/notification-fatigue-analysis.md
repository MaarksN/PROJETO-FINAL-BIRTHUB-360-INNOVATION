# Análise de Fadiga de Notificação

## O Problema no Contexto B2B
Agências de marketing e donos de empresas já lidam com dezenas de ferramentas ruidosas (Slack, WhatsApp, CRMs). Se o BirthHub 360 enviar um alerta push no celular do Gestor toda vez que o Agente de IA responder "Bom dia" a um cliente, o Gestor vai silenciar ou desinstalar o app na primeira hora.

## Tipos que geram mais Unsubscribe/Ignore (A Evitar)
1. **"Status de IA Saudável":** E-mails semanais vazios apenas para dizer "Tudo está funcionando". (Média de abertura < 10%).
2. **Micro-Eventos:** "Seu agente completou 1 chat". (Fadiga extrema).
3. **Alertas de "Novo Lead" sem contexto:** Se o Agente qualifica leads o dia inteiro, o alerta deve ir para o CRM (HubSpot), não para o e-mail do Gestor. E-mail só deve ser usado se o lead for de altíssimo valor ou pedir intervenção humana urgente (Human Handoff).

## O Padrão Ouro (Actionable Notifications)
Para combater a fadiga, uma notificação só deve existir se a resposta para "O que eu faço com isso?" for uma ação clara.
- *Exemplo de boa notificação:* "Atenção: Seu limite de tokens chega a 90% hoje. Clique aqui para aprovar o upgrade automático e não pausar seus agentes."
