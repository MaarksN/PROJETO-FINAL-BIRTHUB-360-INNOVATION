# Política de Versionamento Interno

Os pacotes no diretório `packages/` seguirão [Semantic Versioning (SemVer)](https://semver.org/).

## Incremento de Versões
- Major: Mudanças que quebram a API.
- Minor: Novas funcionalidades retrocompatíveis.
- Patch: Correções de bugs retrocompatíveis.

Utilizaremos Changesets para automatizar o bump de versões nos pacotes e aplicações.
